## Projected Modular Schedule

*This schedule summarizes the planned modular workshop structure. Modules are stand-alone and may be revised as the curriculum is tested.*

<table>
  <thead>
    <tr>
      <th>Day</th>
      <th>Session Topic</th>
      <th>Concept Focus</th>
      <th>Practice / Challenge Focus</th>
    </tr>
  </thead>

    <tr>
      <th colspan="4">Module 1: Robot Systems and Movement</th>
    </tr>
    <tr>
      <td><strong>Day 1</strong></td>
      <td><strong>Introduction to Robotics</strong></td>
      <td>Robotics as an interdisciplinary system: mechanical, electrical, programming, sensors, design, and problem-solving.</td>
      <td>Robot design drawing activity with increasing constraints. Students share different design solutions.</td>
    </tr>
    <tr>
      <td><strong>Day 2</strong></td>
      <td><strong>Movement</strong></td>
      <td>Actuators, robot body awareness, movement commands, speed, timing, and turning.</td>
      <td>Blind Mouse activity and basic mBot movement testing.</td>
    </tr>
    <tr>
      <td><strong>Day 3 / Challenge</strong></td>
      <td><strong>Movement Challenge</strong></td>
      <td>Movement accuracy, path planning, testing, and observation.</td>
      <td>Navigate a simple maze or path. Returning students improve accuracy or attempt harder paths.</td>
    </tr>

</tr>
<tr>
  <td><strong>Day 1</strong></td>

  <td>Robots use sensors to collect information from the world.</td>

<tr>
  <td><strong>Day 2</strong></td>
  <td><strong>Robot Brain</strong></td>

</tr>
<tr>
  <td><strong>Day 3 / Challenge</strong></td>
  <td><strong>See the World Challenge</strong></td>
  <td>Sensor data can guide robot decisions.</td>
  <td>Stop-at-distance, obstacle reaction, or sensor-based maze behavior.</td>
</tr>

<tr>
  <th colspan="4">Module 3: Logic and Autonomy</th>
</tr>
<tr>
  <td><strong>Day 1</strong></td>
  <td><strong>Sensors II</strong></td>
  <td>Sensors convert physical changes into information.</td>
  <td>Blind Mouse with limited sensing and DIY pressure/flex sensor demonstration.</td>
</tr>
<tr>
  <td><strong>Day 2</strong></td>
  <td><strong>Logic and Autonomy</strong></td>
  <td>Robots use conditions, loops, and rules to make decisions.</td>
  <td>Paper logic or pseudocode activity for robot behavior.</td>
</tr>
<tr>
  <td><strong>Day 3 / Challenge</strong></td>
  <td><strong>Line-Following Challenge</strong></td>
  <td>Sensors, movement, and logic work together.</td>
  <td>Line-following challenge using skeleton code and instructor support.</td>
</tr>

<tr>
  <th colspan="4">Module 4: Debugging and Constraints</th>
</tr>
<tr>
  <td><strong>Day 1</strong></td>
  <td><strong>Debugging and Coding</strong></td>
  <td>Syntax errors, logic errors, functions, variables, loops, and conditions.</td>
  <td>Team debugging match using skeleton code or pseudocode.</td>
</tr>
<tr>
  <td><strong>Day 2</strong></td>
  <td><strong>Constraints and Debugging II</strong></td>
  <td>Robot problems can come from code, sensors, motors, power, environment, or mechanical limits.</td>
  <td>“What’s Wrong, Doc?” diagnostic activity and constraint redesign discussion.</td>
</tr>
<tr>
  <td><strong>Day 3 / Challenge</strong></td>
  <td><strong>Challenge In Progress</strong></td>
  <td>Debugging, constraints, or system diagnosis.</td>
  <td>Possible debugging gauntlet, constraint redesign challenge, or fix-the-robot lab.</td>
</tr>

<tr>
  <th colspan="4">Module 5: Robot Mission Lab (In Development)</th>
</tr>
<tr>
  <td><strong>Day 1</strong></td>
  <td><strong>Mission Planning</strong></td>
  <td>Robot missions can be broken into smaller actions, decisions, and tests.</td>
  <td>Students plan a guided mission path, identify actions, and write simple pseudocode.</td>
</tr>
<tr>
  <td><strong>Day 2</strong></td>
  <td><strong>Build, Test, Improve</strong></td>
  <td>Engineers improve robot behavior through testing, evidence, and revision.</td>
  <td>Students test the mission, debug problems, and improve movement, sensor use, or logic.</td>
</tr>
<tr>
  <td><strong>Day 3 / Challenge</strong></td>
  <td><strong>Robot Mission Challenge</strong></td>
  <td>Movement, sensing, logic, debugging, and teamwork come together in one mission.</td>
  <td>Guided mission for new students. Returning students add constraints, accuracy goals, or extra behaviors.</td>
</tr>
